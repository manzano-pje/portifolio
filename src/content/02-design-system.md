# Design System Specification

Projeto: Portfólio Profissional — Paulo Manzano

Versão: 1.0

---

# 1. Objetivo

Este documento define todas as diretrizes visuais da Landing Page.

A imagem anexada ao projeto é a principal referência para o layout.

As regras descritas neste documento complementam essa referência e deverão ser seguidas durante toda a implementação.

O objetivo é criar uma interface moderna, elegante, organizada e profissional, transmitindo confiança desde o primeiro contato.

---

# 2. Conceito Visual

O design deve transmitir a sensação de um software corporativo moderno.

A página não deve parecer um template gratuito nem um portfólio estudantil.

Os conceitos que devem guiar todas as decisões são:

• Elegância

• Organização

• Simplicidade

• Legibilidade

• Sofisticação

• Minimalismo

• Clareza

Todo elemento visual deve possuir uma função.

Não utilizar componentes apenas para preencher espaço.

---

# 3. Referências

A identidade visual deve ser inspirada em produtos modernos como:

- Vercel
- Stripe
- Linear
- Railway
- Supabase

Essas referências servem apenas como inspiração para qualidade visual.

O layout principal continua sendo a imagem fornecida.

---

# 4. Tema

Utilizar exclusivamente Dark Theme.

Não implementar versão Light.

O fundo deve possuir profundidade, porém permanecer discreto.

Evitar gradientes exagerados.

Evitar fundos chamativos.

O conteúdo deve ser o elemento principal da página.

---

# 5. Paleta de Cores

A paleta deve utilizar tons neutros.

Priorizar:

Slate

Gray

Zinc

Utilizar uma única cor de destaque.

Sugestão:

Blue

ou

Cyan

Essa cor deverá aparecer apenas em:

- links
- botões principais
- indicadores
- pequenos detalhes

Nunca utilizar muitas cores simultaneamente.

---

# 6. Tipografia

Fonte principal:

Inter

Toda a página deverá utilizar a mesma família tipográfica.

Hierarquia:

H1

Maior destaque da página.

H2

Título das seções.

H3

Títulos de cards.

Body

Textos descritivos.

Caption

Informações secundárias.

Evitar excesso de pesos diferentes.

A tipografia deve transmitir organização.

---

# 7. Espaçamento

Toda a página deve possuir respiro.

Priorizar espaços amplos.

Utilizar um sistema consistente.

Exemplo:

8px

16px

24px

32px

48px

64px

96px

Não utilizar espaçamentos aleatórios.

---

# 8. Grid

A Landing Page deverá utilizar grid consistente.

Desktop

Conteúdo centralizado.

Tablet

Reorganização gradual.

Mobile

Empilhamento vertical.

Evitar quebras bruscas.

---

# 9. Cards

Todos os cards deverão seguir o mesmo padrão.

Características:

- bordas arredondadas
- fundo discreto
- excelente contraste
- padding consistente
- sombra extremamente suave

Os cards deverão parecer parte do mesmo Design System.

---

# 10. Botões

Existirão apenas dois estilos.

Botão Primário

Cor de destaque.

Maior contraste.

Botão Secundário

Outline ou fundo discreto.

Todos os botões devem possuir:

Hover

Focus

Transição suave

Cursor adequado

Nunca utilizar animações exageradas.

---

# 11. Ícones

Utilizar biblioteca consistente.

Sugestão:

Lucide

Todos os ícones devem possuir o mesmo estilo.

Evitar misturar bibliotecas.

---

# 12. Bordas

Bordas suaves.

Border Radius consistente.

Evitar componentes totalmente quadrados.

Evitar arredondamento exagerado.

---

# 13. Glassmorphism

Utilizar apenas quando fizer sentido.

Características:

- transparência discreta
- blur leve
- excelente contraste

Nunca transformar a Landing Page em uma interface "glass".

O efeito deve ser quase imperceptível.

---

# 14. Sombras

Sombras extremamente discretas.

Elas devem apenas criar profundidade.

Jamais chamar atenção.

---

# 15. Microinterações

Todas as animações devem ser discretas.

Permitido:

Hover

Fade

Scroll Reveal

Opacity

Scale muito pequeno

Transições suaves

Não utilizar:

Bounce

Zoom exagerado

Rotação

Partículas

Explosões

Animações chamativas.

---

# 16. Responsividade

A Landing Page deverá manter a mesma identidade visual em:

Desktop

Tablet

Mobile

Não remover seções.

Não alterar hierarquia.

Apenas reorganizar os elementos.

---

# 17. Imagens

Todas as imagens devem possuir excelente qualidade.

Nunca utilizar imagens pixelizadas.

Caso alguma imagem não exista inicialmente, utilizar placeholders elegantes.

---

# 18. Consistência

Todos os componentes devem parecer pertencentes ao mesmo sistema.

O visitante nunca deverá perceber mudanças de estilo entre as seções.

A Landing Page deve transmitir unidade visual.

---

# 19. O que evitar

Não utilizar:

Bootstrap

Vuetify

Material Design

Visual Gamer

Neon

Glow exagerado

Gradientes fortes

Cores saturadas

Carrosséis

Canvas

Three.js

Lottie

Partículas

Efeitos 3D

Parallax exagerado

Excesso de animações

Templates prontos

Visual infantil

---

# 20. Critérios de Aceitação

O Design System será considerado aprovado quando:

• Toda a página apresentar identidade visual consistente.

• A leitura for confortável.

• O conteúdo for o protagonista.

• O layout transmitir profissionalismo.

• A navegação for intuitiva.

• A experiência for consistente em Desktop, Tablet e Mobile.

• O visitante perceber qualidade sem que os efeitos visuais chamem mais atenção que o conteúdo.

---

# Observação Final

A imagem anexada possui prioridade sobre qualquer interpretação visual.

Caso exista conflito entre este documento e a imagem, a estrutura visual da imagem deverá ser preservada, adaptando apenas o necessário para garantir responsividade, acessibilidade e boas práticas de desenvolvimento.
